## RC4暗号について

### RC4の仕組み

RC4は1987年に**Ron Rivest**により設計されたストリーム暗号です。シンプルな設計で高速なため、WEP・WPA・TLS/SSLなど多くの通信プロトコルで採用されました。

#### RC4のアルゴリズム構成

RC4は2つのフェーズで構成されています：

**フェーズ1: KSA（Key Scheduling Algorithm）**

秘密鍵から初期状態ベクトル（S-box）を生成します。

```
入力: 秘密鍵 K (可変長, 通常40-256 bits)
出力: 状態ベクトル S = [0, 1, 2, ..., 255]

手順:
1. S を初期化: S[i] = i (i = 0 to 255)
2. j = 0
3. i = 0 to 255 について以下を実行:
   j = (j + S[i] + K[i mod len(K)]) mod 256
   S[i] と S[j] をスワップ
```

**フェーズ2: PRGA（Pseudo-Random Generation Algorithm）**

KSAで生成された状態ベクトルから鍵ストリームを生成します。

```
入力: 状態ベクトル S（KSAの出力）
出力: 鍵ストリーム z (無限に生成可能)

手順（各バイト生成時）:
1. i = (i + 1) mod 256
2. j = (j + S[i]) mod 256
3. S[i] と S[j] をスワップ
4. t = (S[i] + S[j]) mod 256
5. 出力: K = S[t]
```

#### 暗号化と復号化

RC4の暗号化は **XOR演算** に基づいています：

```
暗号文 C = 平文 P ⊕ 鍵ストリーム K
復号文 P = 暗号文 C ⊕ 鍵ストリーム K
```

XOR演算の逆演算性により、鍵ストリームがあれば復号は自動的に成功します。

### 既知の脆弱性

RC4には複数の既知脆弱性があります：

| 脆弱性 | 詳細 | 影響 |
|------|------|------|
| **FMS攻撃** | 初期バイトの統計的弱点を利用 | WEPの破壊 |
| **初期バイトバイアス** | 鍵ストリーム最初のバイトの偏り | 平文推測可能 |
| **第2バイトバイアス** | 本プロジェクトで検証 | 平文復号可能 |
| **Long-RC4弱点** | 長いメッセージでの相関 | 情報漏洩 |

### スクリプト1: `rc4_keystream_bias_analysis.py`

**目的**：RC4が生成する鍵ストリームに存在する統計的バイアスを可視化

#### スクリプトが行うこと

1. 100万回の実験を実行
2. 毎回異なるランダムな秘密鍵を生成
3. RC4アルゴリズムで鍵ストリームの最初の3バイトを生成
4. 各バイト位置（1, 2, 3番目）の値の出現頻度をカウント
5. 理論値（均等分布の期待値）との比較をグラフで可視化

#### 数学的背景

理想的なストリーム暗号は **均等分布** を示すべきです：

```
期待値（理論値）= 1,000,000 回 / 256 値 = 3,906.25 回

各値の出現回数 ~ N(μ=3,906.25, σ²=...)
```

RC4の場合、第2バイトは統計的に有意な偏りを示します：

```
値「0」の出現回数: 7,915 回（理論値比: 約2.03倍！）
```

#### 簡略化について

本スクリプトは簡略化されています：

- **実際のRC4**: 毎回異なるIV（初期化ベクトル）を使用せず、秘密鍵のみで初期化
- **本スクリプト**: より現実的なWEP攻撃シナリオを再現するため、ランダムIV（3バイト）+ 固定秘密鍵を使用
- **実装**: フルRC4の検証は省略し、KSA→PRGA→バイアス分析に特化

#### 実行方法

### スクリプト2: `rc4_statistical_decryption_attack.py`

**目的**：複数の暗号化パケットから秘密鍵を知らずに平文を統計的に推測する、4つの復号化攻撃を実装・比較

#### スクリプトが行うこと

4種類の攻撃手法を順序に実装：

**攻撃1: 第1バイト分析（100k サンプル）**

- 100,000個の異なるランダム秘密鍵で同じ平文を暗号化
- 暗号文バイトの出現頻度から平文を推測
- 結果: 失敗（成功率 0.45%）

```python
cipher_byte = target_plain_byte ^ keystream_byte
```

**攻撃2: 第1バイト分析（1000万 サンプル）**

- 攻撃1と同じだが、サンプル数を100倍に増加
- 結果: 失敗（成功率 0.40%）

**攻撃3: 第2バイト分析（FMS攻撃）** - 成功

- 第2バイト目を分析（第1バイトよりも強い偏りがある）
- 100,000サンプルで検証
- 結果: 成功（成功率 77%）

**攻撃4: 複数バイト同時分析**

- 複数バイトの組み合わせ（"ABC" = [65, 66, 67]）を同時に分析
- 複数バイトの統計的相関を利用
- 結果: 失敗（成功率 0.0%）

#### 数学的背景

##### 統計的推測の原理

暗号文バイトは平文 **P** と鍵ストリーム **K** のXOR結果：

```
C = P ⊕ K

P を固定した複数回の暗号化:
C₁ = P ⊕ K₁
C₂ = P ⊕ K₂
C₃ = P ⊕ K₃
...
```

**K_i がランダムに均等分布している場合**、C_i も均等分布するはず。

**しかし、RC4の第2バイトに偏りがある場合**：

```
K₂ の分布が偏っている
    ↓
C₂ = P ⊕ K₂ も偏った分布になる
    ↓
C₂ の出現頻度が高い値を逆算すれば、
そのバイトが正解の平文である可能性が高い
```

##### FMS攻撃の数学

平文 P が正しい場合、暗号文の出現頻度分布が最大になる：

```
L(P) = Σ count(C_i)  ここで C_i = P ⊕ K_i

正解の P に対して L(P) が最大値を示す
```

#### 簡略化について

本スクリプトは簡略化されています：

- **実際のFMS攻撃**: 秘密鍵の関係性を統計的に追跡し、複数バイト間の相関を利用
- **本スクリプト**: 各バイト独立に分析（相関分析なし）
- **実装**: 単純な度数分布から最頻値を選択（より高度な統計手法なし）
- **サンプル数**: 現実のWEP攻撃は数千～数百万パケット必要、本実装は簡潔性重視

## 統計的有意性検定

### Jupyter Notebook: `rc4_statistical_significance_test.ipynb`

**目的**：カイ二乗検定を用いて、RC4のバイアスが統計的に有意であることを厳密に検証

#### スクリプトが行うこと

このJupyterノートブックは以下の分析を実装しています：

1. **RC4シミュレーション**
   - 100万回の異なるランダム鍵を使用して鍵ストリームを生成
   - 各バイト位置（1, 2, 3番目）の値の出現頻度をカウント

2. **カイ二乗検定**
   - 観測度数と期待度数の比較
   - 各バイト位置で帰無仮説（均等分布）を検定
   - p値を計算し、有意水準α=0.05で判定

3. **詳細分析**
   - 各バイト値の相対偏差を計算
   - 最も偏りが大きい値を抽出
   - KL距離（Kullback-Leibler divergence）で分布差を測定

4. **可視化**
   - カイ二乗統計量の棒グラフ
   - p値のプロット（対数スケール）
   - 有意性の判定を色分け表示

#### 初心者向けコメント

ノートブックには以下の特徴があります：

- **日本語ラベルなし**: matplotlibで日本語表示の問題を回避し、すべて英語で表示
- **詳細なコメント**: 各ステップで何を行っているのか、なぜそうするのかを説明
- **段階的構成**: 8つのセルに分割し、順序に実行できる構造

#### カイ二乗検定の数学

**帰無仮説と対立仮説**

```
H₀ (帰無仮説): RC4の出力は均等分布である
H₁ (対立仮説): RC4の出力は均等分布ではない

有意水準: α = 0.05 (5%)
```

**カイ二乗統計量**

```
χ² = Σ [(観測度数 - 期待度数)² / 期待度数]

各バイト値について計算し、すべて合計
χ² が大きいほど、観測分布が均等分布から遠い
```

**p値と判定**

```
p値 = P(χ² ≥ 観測値 | H₀が真)

p値 < 0.05 → H₀を棄却 → 有意な偏りあり（赤色）
p値 ≥ 0.05 → H₀を採択 → 有意な偏びなし（青色）
```

```bash

```

または、JupyterLabから直接開き、セルを上から順に実行してください。

**KLダイバージェンス（乱らしさの距離）**：

KLダイバージェンスは、「生成された数が、完璧にランダムな状態からどれだけズレているか」を測る指標です。完璧にランダムなら0、ズレが大きいほど数値が大きくなります。

| バイト | KL値 | 説明 |
|------|------|------|
| バイト 1 | 0.00020344 | ちょっと程度のズレ |
| バイト 2 | 0.00161309 | **結構なズレが見られます** |
| バイト 3 | 0.00014712 | ほぼランダムに近い |

**何が起きているのか**：
- **バイト 1**: 統計的にはズレが検出されました。でも複数の値に偏りが分散しているので、攻撃には使いづらいです。
- **バイト 2**: これが本当に問題です！値の0が期待値の約2倍も出現しています。このズレの大きさこそが、FMS攻撃が成功する理由です。
- **バイト 3**: ほぼ均等分布で、特に問題はないように見えます。

**結論**: 実験の結果、**バイト 2 が圧倒的に強い偏りを持っている**ことが分かりました。これはRC4の設計的な弱点です。

### なぜ第2バイトが成功するのか

RC4のKSAとPRGAの相互作用により、**第2バイト目に統計的バイアスが集中**します：

```
数学的理由:
- KSA後の状態ベクトル S は秘密鍵に依存
- PRGA第1ステップで i=1 のため、S[1] がアクセスされやすい
- 最初の状態初期化の影響が残存
- 結果: 第2バイトの値「0」が約2倍の頻度で出現
```

### 攻撃の実行フロー

```
1. 複数の異なる秘密鍵で同じ平文を暗号化
   暗号文群: {C₁, C₂, C₃, ..., C_n}

2. 各暗号文バイトの出現頻度をカウント
   度数分布: {値0: 100回, 値1: 99回, ..., 値2: 769回}

3. 最も出現頻度が高いバイト値を抽出
   最頻値 = 2 (出現769回)

4. 統計的推測により平文を復元
   推測値 = 2  <- これが正解の平文バイト！
```

## 実験結果と数学的解釈

### キーストリームバイアス分析の結果

#### グラフ解釈

`rc4_keystream_bias_analysis.py` が出力する3つのグラフ：

1. **第1バイト分布**
   - ほぼ均等分布
   - 理論値（赤い破線）との乖離は統計的ノイズレベル
   - 攻撃困難

2. **第2バイト分布** - 異常を検出
   - 値「0」が極度に突出
   - 理論値の**約2倍**の頻度
   - 攻撃成功の鍵

3. **第3バイト分布**
   - ほぼ均等分布
   - 第1バイトと同様

#### 統計的有意性

カイ二乗検定により、第2バイトの偏りは**統計的に有意**です：

```
帰無仮説 H₀: RC4の出力は均等分布
対立仮説 H₁: RC4の出力は非均等分布

観測度数: 実験から得られた各バイト値の頻度
期待度数: 1,000,000 / 256 ≈ 3,906.25

χ² = Σ (観測度数 - 期待度数)² / 期待度数

-> χ² 値は自由度255のカイ二乗分布で極度に高い値
-> H₀ を棄却、第2バイトに有意な偏りあり
```

### 復号化攻撃の結果

#### 攻撃成功メカニズム

攻撃3（第2バイトFMS）の成功理由：

```
数値例:
正解平文バイト = 65 ('A')

1,000万回中、異なる秘密鍵で暗号化:

K₁: 鍵ストリーム第2バイト = 234 -> 暗号文 = 65 ⊕ 234 = 179
K₂: 鍵ストリーム第2バイト = 10  -> 暗号文 = 65 ⊕ 10 = 75
K₃: 鍵ストリーム第2バイト = 0   -> 暗号文 = 65 ⊕ 0 = 65 <- 値0が出現しやすい！
...（多数）

結果: 暗号文「65」が他より高頻度で出現
     -> 平文 = 65 と推測成功！
```
